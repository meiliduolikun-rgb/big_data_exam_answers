## 1. Что такое LSH?

中文题目：

```text
什么是局部敏感哈希？
```

### 导师讲解

这个问题是整门课最核心的问题之一。老师想看你是否理解：

```text
大数据中为什么不能直接两两比较所有对象？
如何快速找到相似对象？
```

假设有 100 万篇文章，如果每两篇都比较一次，相当于接近：

```text
1000000 × 1000000
```

这个计算量太大。所以我们需要一种近似方法，先把可能相似的对象找出来，再做精确比较。

LSH 的思想就是：

```text
相似对象更容易进入同一个 bucket。
不相似对象不容易进入同一个 bucket。
```

### 知识点解释

LSH 全称：

```text
Locality Sensitive Hashing
```

俄语：

```text
локально-чувствительное хэширование
```

普通 hash 的目标通常是让数据分布均匀；LSH 的目标是保留相似性。

普通 hash：

```text
相似对象也可能被分到完全不同的位置。
```

LSH：

```text
相似对象高概率进入同一个 bucket。
```

LSH 常用于：

```text
相似文档查找
去重
图片相似搜索
推荐系统
近似最近邻搜索
```

### 俄语正式回答

Локально-чувствительное хэширование, или LSH, — это семейство хэш-функций, которое используется для приближенного поиска похожих объектов в больших данных. Главная идея состоит в том, что похожие объекты с высокой вероятностью попадают в один и тот же bucket, а непохожие объекты попадают в один bucket с меньшей вероятностью.

LSH позволяет не сравнивать каждый объект со всеми остальными объектами. Сначала мы с помощью хэширования получаем небольшое число кандидатов на похожие объекты, а затем сравниваем только эти кандидаты более точно.

LSH применяется для поиска похожих документов, удаления дубликатов, рекомендательных систем, поиска похожих изображений и приближенного поиска ближайших соседей.

### 中文对照

局部敏感哈希 LSH 是用于大数据中近似查找相似对象的一类哈希函数。它的核心思想是，相似对象有较高概率进入同一个 bucket，不相似对象进入同一个 bucket 的概率较低。

LSH 可以避免把每个对象和所有对象都比较。系统先用哈希找到少量候选相似对象，然后只对这些候选对象做更精确的比较。

LSH 可用于相似文档查找、去重、推荐系统、相似图片搜索和近似最近邻搜索。

### 和项目/大数据任务的关系

你的 Lenta.ru 项目没有直接使用 LSH，但它也属于“相似搜索”任务：

```text
用户输入 query
系统查找相似新闻
```

你的项目用的是：

```text
Word2Vec + BM25
```

而 LSH 是另一种可以加速相似搜索的方法。  
如果数据量更大，可以先用 LSH 找候选新闻，再用 Word2Vec/BM25 精排。

## 2. Что такое MinHash и сходство по Жаккару?

中文题目：

```text
什么是 MinHash 和 Jaccard 相似度？
```

### 导师讲解

这个问题通常和 LSH 一起考。老师想看你是否知道：

```text
集合之间怎么定义相似？
MinHash 为什么能近似 Jaccard 相似度？
```

文本可以看成词集合。例如：

```text
A = {футбол, чемпионат, россия}
B = {футбол, матч, россия}
```

它们有共同词，所以相似。

### 知识点解释

Jaccard similarity：

```text
J(A, B) = |A ∩ B| / |A ∪ B|
```

交集越大，并集越小，相似度越高。

Jaccard distance：

```text
D(A, B) = 1 - J(A, B)
```

MinHash 是一种估计 Jaccard 相似度的方法。它的核心性质：

```text
P(hmin(A) = hmin(B)) = J(A, B)
```

也就是说：

```text
两个集合 MinHash 值相同的概率等于它们的 Jaccard 相似度。
```

### 俄语正式回答

Сходство по Жаккару используется для сравнения множеств. Оно определяется как отношение размера пересечения множеств к размеру их объединения:

```text
J(A, B) = |A ∩ B| / |A ∪ B|
```

Расстояние по Жаккару равно:

```text
D(A, B) = 1 - J(A, B)
```

MinHash — это метод локально-чувствительного хэширования, который позволяет приближенно оценивать сходство по Жаккару без полного вычисления пересечения и объединения множеств.

Для случайной перестановки элементов значение `hmin(S)` множества `S` — это первый элемент множества в этой перестановке. Главное свойство MinHash:

```text
P(hmin(A) = hmin(B)) = J(A, B)
```

На практике используется несколько независимых хэш-функций, которые образуют сигнатуру множества:

```text
Hmin(S) = [h1(S), h2(S), ..., hk(S)]
```

Сходство двух множеств оценивается как доля совпадающих элементов их сигнатур.

### 中文对照

Jaccard 相似度用于比较集合，定义为两个集合交集大小除以并集大小。Jaccard 距离等于 1 减去 Jaccard 相似度。

MinHash 是一种局部敏感哈希方法，它可以不用完整计算交集和并集，就近似估计 Jaccard 相似度。

对于某个随机排列，`hmin(S)` 是集合 `S` 中在该排列下最先出现的元素。MinHash 的关键性质是：两个集合 MinHash 值相等的概率等于它们的 Jaccard 相似度。

实际中会用多个哈希函数形成签名，再通过签名相同位置的比例估计相似度。

### 和项目/大数据任务的关系

你的 Lenta.ru 项目使用 Word2Vec + BM25，没有直接用 MinHash。  
但如果把新闻表示成词集合，就可以用 Jaccard/MinHash 做：

```text
相似新闻检测
重复新闻去重
近似文本搜索
```

在其他大数据任务中，MinHash 经常用于：

```text
反抄袭
网页去重
搜索引擎索引优化
相似用户查找
```

## 3. Что такое MinHashLSH и BucketedRandomProjectionLSH в PySpark?

中文题目：

```text
PySpark 中的 MinHashLSH 和 BucketedRandomProjectionLSH 是什么？
```

### 导师讲解

老师不仅想听理论，还想知道你是否知道实际工具。  
这个问题重点是：

```text
不同类型的数据应该用不同的 LSH 工具。
```

如果数据是集合，用 MinHashLSH。  
如果数据是数值向量，用 BucketedRandomProjectionLSH。

### 知识点解释

`MinHashLSH`：

```text
适合集合
适合 Jaccard distance
常用于文本去重、相似文档查找
```

`BucketedRandomProjectionLSH`：

```text
适合数值向量
适合 Euclidean distance
用随机投影把向量放进 bucket
```

### 俄语正式回答

В PySpark есть готовые инструменты для локально-чувствительного хэширования: `MinHashLSH` и `BucketedRandomProjectionLSH`.

`MinHashLSH` используется для объектов, которые представлены как множества или разреженные векторы. Он основан на сходстве по Жаккару и подходит для поиска похожих документов, удаления дубликатов и сравнения множеств признаков.

`BucketedRandomProjectionLSH` используется для числовых векторов и Евклидова расстояния. Метод основан на случайной проекции: векторы проецируются на случайное направление и затем попадают в buckets. Близкие векторы с большей вероятностью попадают в один bucket.

Таким образом, если данные представлены как множества, лучше использовать `MinHashLSH`. Если данные представлены как числовые векторы и используется Евклидово расстояние, лучше использовать `BucketedRandomProjectionLSH`.

### 中文对照

PySpark 提供了两个常用 LSH 工具：`MinHashLSH` 和 `BucketedRandomProjectionLSH`。

`MinHashLSH` 用于集合或稀疏向量，基于 Jaccard 相似度，适合相似文档查找和去重。

`BucketedRandomProjectionLSH` 用于数值向量和欧几里得距离。它基于随机投影，把向量投影到随机方向并分配到 bucket。距离近的向量更可能进入同一个 bucket。

所以，集合数据用 MinHashLSH，数值向量用 BucketedRandomProjectionLSH。

### 和项目/大数据任务的关系

你的项目中新闻可以有两种表示：

```text
词集合：可以用 MinHashLSH
Word2Vec 向量：可以用 BucketedRandomProjectionLSH
```

目前你的项目直接用 Word2Vec 余弦相似度和 BM25 排序。  
如果以后数据量更大，可以用 LSH 做第一阶段召回：

```text
LSH 找候选新闻
Word2Vec + BM25 精确排序
```

## 4. Что такое HDFS?

中文题目：

```text
什么是 HDFS？
```

### 导师讲解

HDFS 是大数据课程中最基础的概念之一。  
老师想看你是否理解：

```text
为什么大数据不能只放在普通本地文件系统？
Hadoop 如何保存大文件？
```

### 知识点解释

HDFS 全称：

```text
Hadoop Distributed File System
```

核心组件：

```text
NameNode：保存元数据
DataNode：保存真实数据块
Block：文件被切成的块
Replication：副本机制
```

特点：

```text
适合大文件
适合顺序读取
支持容错
不适合大量小文件
不适合频繁修改
```

### 俄语正式回答

HDFS, или Hadoop Distributed File System, — это распределенная файловая система Hadoop, предназначенная для хранения больших файлов на кластере машин.

В HDFS файл делится на блоки. Эти блоки распределяются по разным узлам кластера. `NameNode` хранит метаданные: имена файлов, структуру каталогов и информацию о том, где находятся блоки. `DataNode` хранит сами блоки данных.

Для отказоустойчивости HDFS использует репликацию: один и тот же блок может храниться на нескольких DataNode. Если один узел выходит из строя, данные можно прочитать с другой копии.

HDFS хорошо подходит для больших файлов и последовательного чтения, но плохо подходит для большого числа маленьких файлов и частых изменений данных.

### 中文对照

HDFS 是 Hadoop 的分布式文件系统，用于在多台机器组成的集群上保存大文件。

在 HDFS 中，文件会被切成 block，这些 block 分布在不同节点上。NameNode 保存元数据，比如文件名、目录结构和 block 位置。DataNode 保存真实数据块。

HDFS 通过副本机制实现容错。如果某个节点坏了，可以从其他副本读取数据。

HDFS 适合大文件和顺序读取，不适合大量小文件和频繁修改。

### 和项目/大数据任务的关系

你的 Lenta.ru 项目使用 HDFS 保存：

```text
原始新闻数据：/datasets/news_lenta.csv
Word2Vec 模型：/MeiliDuolikun/lenta_search/models/w2v_100k_v2
文档索引：/MeiliDuolikun/lenta_search/index/docs_100k_v2_text
BM25 统计：/MeiliDuolikun/lenta_search/stats/bm25_100k_v2
```

这说明项目不是只在本地文件上运行，而是使用了 Hadoop 的分布式存储。

## 5. Что такое MapReduce?

中文题目：

```text
什么是 MapReduce？
```

### 导师讲解

MapReduce 是大数据处理的经典模型。老师想看你是否理解：

```text
怎么把一个大任务拆开并行处理？
怎么把中间结果合并？
```

### 知识点解释

MapReduce 分为三个核心阶段：

```text
Map
Shuffle
Reduce
```

Map：

```text
处理输入数据，生成中间 key-value 对。
```

Shuffle：

```text
按 key 把中间结果分组。
```

Reduce：

```text
对同一个 key 的值进行聚合。
```

经典例子：Word Count。

```text
mapper: word -> (word, 1)
reducer: sum counts for each word
```

### 俄语正式回答

MapReduce — это модель распределенной обработки больших данных. Она позволяет разбить большую задачу на независимые части, обработать их параллельно, а затем объединить результаты.

MapReduce состоит из трех основных этапов. На этапе `Map` входные данные обрабатываются и преобразуются в промежуточные пары ключ-значение. На этапе `Shuffle` данные группируются по ключу. На этапе `Reduce` значения с одинаковым ключом агрегируются и формируется итоговый результат.

Классический пример — подсчет слов. Mapper для каждого слова выводит пару `(word, 1)`, а reducer суммирует все значения для одного и того же слова.

MapReduce хорошо подходит для пакетной обработки больших данных, потому что задачи можно выполнять параллельно на разных узлах кластера.

### 中文对照

MapReduce 是一种分布式处理大数据的模型。它把大任务拆成独立部分并行处理，然后合并结果。

Map 阶段处理输入数据并生成中间 key-value 对。Shuffle 阶段按 key 分组。Reduce 阶段聚合同一个 key 的值，形成最终结果。

经典例子是词频统计：mapper 输出 `(word, 1)`，reducer 对每个词求和。

MapReduce 适合大规模批处理任务，因为不同任务可以在集群节点上并行执行。

### 和项目/大数据任务的关系

你的项目选择的是 Spark in Hadoop，所以没有单独写 `mapper.py` 和 `reducer.py`。但 Spark 中也有类似思想：

```text
filter/select/transform 类似 mapper
groupBy/agg/orderBy/limit 类似 reducer
```

你的项目中：

```text
文本清洗、分词、向量化 = mapper-like
BM25 统计、排序 Top-10 = reducer-like
```

## 6. Что такое Hadoop Streaming?

中文题目：

```text
什么是 Hadoop Streaming？
```

### 导师讲解

这个问题考察实践能力。老师想知道你是否知道：

```text
除了 Java MapReduce，还能怎么写 MapReduce？
```

Hadoop Streaming 允许你用 Python 这种脚本语言写 mapper/reducer。

### 知识点解释

Hadoop Streaming 的特点：

```text
mapper 从 stdin 读数据
mapper 向 stdout 写结果
reducer 从 stdin 读 mapper 输出
reducer 向 stdout 写最终结果
```

命令形式类似：

```bash
hadoop jar hadoop-streaming.jar \
  -mapper mapper.py \
  -reducer reducer.py \
  -input input_path \
  -output output_path
```

老师课上还提到 typed bytes / hadoop-streaming-tb。

### 俄语正式回答

Hadoop Streaming — это инструмент Hadoop, который позволяет писать mapper и reducer не только на Java, но и на других языках, например Python, Bash или Perl.

В Hadoop Streaming mapper читает данные из стандартного ввода и записывает промежуточный результат в стандартный вывод. Reducer получает сгруппированные данные также через стандартный ввод и записывает итоговый результат в стандартный вывод.

Это удобно, потому что можно быстро реализовать MapReduce-задачу на Python. Кроме обычного текстового формата, могут использоваться key-value пары, SequenceFile и typed bytes.

Пример запуска:

```text
hadoop-streaming -mapper mapper.py -reducer reducer.py -input input -output output
```

### 中文对照

Hadoop Streaming 是 Hadoop 的工具，它允许不只用 Java，也可以用 Python、Bash 等语言写 mapper 和 reducer。

在 Hadoop Streaming 中，mapper 从标准输入读取数据，并把中间结果写到标准输出。reducer 也从标准输入读取分组后的数据，并把最终结果写到标准输出。

它的优点是可以快速用 Python 实现 MapReduce。除了普通文本，也可以使用 key-value、SequenceFile 和 typed bytes 等格式。

### 和项目/大数据任务的关系

你的最终 Lenta 项目没有使用 Hadoop Streaming，而是使用 Spark。  
但你之前图片项目/课堂示例里见过：

```text
hadoop-streaming-tb
tb.read()
tb.write()
```

这就是 typed bytes 风格的 Hadoop Streaming。

在答辩中可以说：

```text
Hadoop Streaming 是一种显式写 mapper/reducer 的方式；
Spark 是另一种分布式处理方式。
```

## 7. Что такое Spark и PySpark?

中文题目：

```text
什么是 Spark 和 PySpark？
```

### 导师讲解

Spark 是你项目真正使用的工具。老师可能会问：

```text
为什么用 Spark？
Spark 和 MapReduce 有什么区别？
```

你要能说明：

```text
Spark 更适合迭代算法和机器学习。
```

### 知识点解释

Spark 是分布式计算框架。

它支持：

```text
RDD
DataFrame
SparkSQL
MLlib
PySpark
```

Spark 相比传统 MapReduce：

```text
支持内存计算
更适合迭代算法
更适合机器学习
API 更方便
```

PySpark：

```text
Spark 的 Python API
```

### 俄语正式回答

Apache Spark — это распределенный фреймворк для обработки больших данных. Он поддерживает RDD, DataFrame, SparkSQL и MLlib. Spark часто используется для аналитики, машинного обучения и итеративных алгоритмов.

В отличие от классического Hadoop MapReduce, Spark может выполнять вычисления в памяти, поэтому он обычно быстрее для итеративных задач. MapReduce после каждого этапа часто записывает данные на диск, а Spark может хранить промежуточные данные в памяти.

PySpark — это Python API для Apache Spark. С его помощью можно обрабатывать DataFrame, выполнять SQL-запросы, обучать модели машинного обучения и работать с большими данными в распределенной среде.

### 中文对照

Apache Spark 是用于处理大数据的分布式计算框架。它支持 RDD、DataFrame、SparkSQL 和 MLlib，常用于数据分析、机器学习和迭代算法。

和传统 Hadoop MapReduce 相比，Spark 支持内存计算，所以在迭代任务中通常更快。MapReduce 每个阶段常写磁盘，而 Spark 可以把中间数据保存在内存中。

PySpark 是 Spark 的 Python API，可以用 Python 处理 DataFrame、执行 SQL 查询、训练机器学习模型和进行分布式大数据处理。

### 和项目/大数据任务的关系

你的 Lenta.ru 项目用 Spark/PySpark 完成：

```text
读取 HDFS 数据
清洗文本
分词
训练 Spark ML Word2Vec
构建 Parquet 索引
计算 BM25 统计
执行查询排序
```

Word2Vec 训练是迭代式机器学习任务，所以用 Spark 比手写 MapReduce 更自然。

## 8. Что такое ACID, BASE и CAP?

中文题目：

```text
什么是 ACID、BASE 和 CAP？
```

### 导师讲解

这个问题是数据库和分布式系统理论。老师想看你是否理解：

```text
传统数据库和分布式 NoSQL 系统的设计目标不同。
```

ACID 偏强一致事务。  
BASE 偏高可用和最终一致。  
CAP 说明分布式系统中不能三者都完美满足。

### 知识点解释

ACID：

```text
Atomicity：原子性
Consistency：一致性
Isolation：隔离性
Durability：持久性
```

BASE：

```text
Basically Available：基本可用
Soft state：软状态
Eventually consistent：最终一致
```

CAP：

```text
Consistency：一致性
Availability：可用性
Partition tolerance：分区容错
```

CAP 定理：

```text
分布式系统不能同时完全满足 C、A、P。
```

### 俄语正式回答

ACID — это набор требований к транзакциям в классических реляционных базах данных. Он включает атомарность, согласованность, изолированность и долговечность.

Атомарность означает, что транзакция выполняется либо полностью, либо не выполняется вообще. Согласованность означает, что база данных переходит из одного корректного состояния в другое. Изолированность означает, что параллельные транзакции не мешают друг другу. Долговечность означает, что после фиксации транзакции данные не теряются.

BASE — это подход, часто используемый в распределенных NoSQL-системах. Он означает базовую доступность, мягкое состояние и согласованность в итоге. Такие системы могут не давать строгой согласованности в каждый момент времени, но со временем приходят к согласованному состоянию.

Теорема CAP утверждает, что распределенная система не может одновременно полностью гарантировать consistency, availability и partition tolerance. На практике при наличии сетевых разделений приходится выбирать компромисс между согласованностью и доступностью.

### 中文对照

ACID 是传统关系数据库事务的要求，包括原子性、一致性、隔离性和持久性。

原子性表示事务要么全部执行，要么完全不执行。一致性表示数据库从一个正确状态变到另一个正确状态。隔离性表示并发事务互不干扰。持久性表示事务提交后数据不会丢失。

BASE 是分布式 NoSQL 系统常见思想，包括基本可用、软状态和最终一致。这类系统不一定每一刻都强一致，但最终会达到一致。

CAP 定理说明分布式系统不能同时完全保证一致性、可用性和分区容错。现实中如果发生网络分区，系统通常要在一致性和可用性之间取舍。

### 和项目/大数据任务的关系

你的项目主要是分析型任务，不是事务型数据库。  
它更接近：

```text
大规模离线分析
HDFS 存储
Spark 批处理
最终生成索引
```

所以它不强调 ACID 事务，而是强调：

```text
分布式存储
可扩展处理
最终生成一致的搜索索引
```

在其他大数据系统中，NoSQL 数据库常为了高可用和扩展性采用 BASE，而不是严格 ACID。

## 9. Что такое Parquet и ORC?

中文题目：

```text
什么是 Parquet 和 ORC？
```

### 导师讲解

这个问题考察大数据存储格式。老师想看你是否理解：

```text
为什么大数据分析不用普通 CSV 就完事？
```

CSV 简单，但分析效率低。  
Parquet/ORC 是列式格式，适合 Spark/Hive 分析。

### 知识点解释

行式存储：

```text
按一整行保存
适合事务和整行读写
```

列式存储：

```text
按列保存
适合分析查询
只读需要的列
压缩效果好
```

Parquet：

```text
列式格式
Spark 常用
适合分析
```

ORC：

```text
列式格式
Hive 常用
压缩好
```

### 俄语正式回答

Parquet и ORC — это колонночные форматы хранения данных, которые используются в экосистеме больших данных.

В строковом хранении данные сохраняются по строкам, что удобно для транзакционных систем и операций с целыми записями. В колонночном хранении данные сохраняются по столбцам. Это удобно для аналитических запросов, потому что часто нужно читать не все поля, а только некоторые столбцы.

Apache Parquet хорошо интегрирован со Spark и Hive. Он поддерживает сжатие, схему данных и чтение только нужных столбцов.

ORC также является колонночным форматом и часто используется вместе с Hive. Он обеспечивает эффективное сжатие и быстрое выполнение аналитических запросов.

По сравнению с CSV, Parquet и ORC обычно эффективнее для больших аналитических задач.

### 中文对照

Parquet 和 ORC 是大数据生态中常用的列式存储格式。

行式存储按行保存数据，适合事务系统和整行操作。列式存储按列保存数据，适合分析查询，因为分析时通常只需要读取部分列。

Parquet 与 Spark/Hive 集成很好，支持压缩、schema 和列裁剪。ORC 也是列式格式，常用于 Hive，压缩率高，适合大表分析。

相比 CSV，Parquet 和 ORC 更适合大规模分析任务。


但训练后生成的索引是 Parquet：

```text
/MeiliDuolikun/lenta_search/index/docs_100k_v2_text
```

这样查询时 Spark 可以更高效地读取需要的字段，比如：


## 10. Что такое PageRank и TF-IDF?

中文题目：

```text
什么是 PageRank 和 TF-IDF？
```

### 导师讲解

这是两个经典算法。PageRank 是图上的重要性排序，TF-IDF 是文本中的关键词权重。

PageRank 常用于网页排名。  
TF-IDF 常用于文本搜索和关键词提取。

### 知识点解释：PageRank

核心思想：

```text
一个网页重要，如果很多重要网页链接到它。
```

网页可以表示成图：

```text
节点 = 网页
边 = 链接
```

带 damping factor 的公式：

```text
PR(u) = (1 - d) / N + d * sum(PR(v) / L(v))
```

其中：

```text
d 通常是 0.85
N 是网页数量
v 是指向 u 的网页
L(v) 是 v 的出链数量
```

### 知识点解释：TF-IDF

TF：

```text
term frequency
词在文档中出现的频率
```

IDF：

```text
inverse document frequency
词在整个语料中越少见，权重越高
```

公式：

```text
idf(t) = log(N / df(t))
tfidf(t, d) = tf(t, d) * idf(t)
```

### 俄语正式回答

PageRank — это алгоритм оценки важности страниц в графе ссылок. Основная идея состоит в том, что страница считается важной, если на нее ссылаются другие важные страницы.

Веб можно представить как ориентированный граф, где вершины — это страницы, а ребра — ссылки между страницами. PageRank вычисляется итеративно и может быть представлен как матричное преобразование.

Формула с damping factor:

```text
PR(u) = (1 - d) / N + d * sum(PR(v) / L(v))
```

TF-IDF используется для выявления ключевых слов и оценки важности термина в документе. TF означает частоту термина в документе, а IDF показывает, насколько редко термин встречается во всей коллекции.

Формула IDF:

```text
idf(t) = log(N / df(t))
```

Формула TF-IDF:

```text
tfidf(t, d) = tf(t, d) * idf(t)
```

Смысл TF-IDF состоит в том, что слово важно для документа, если оно часто встречается в этом документе, но редко встречается во всей коллекции.

### 中文对照

PageRank 是网页链接图上的重要性排序算法。一个网页如果被很多重要网页链接，它也会更重要。网页可以表示成有向图，PageRank 可以通过迭代或矩阵变换计算。

TF-IDF 用于关键词提取和文本检索。TF 表示词在文档中的频率，IDF 表示词在整个语料中的稀有程度。一个词如果在当前文档中经常出现，但在整个语料中不常见，那么它对这篇文档更重要。

### 和项目/大数据任务的关系

你的项目没有使用 PageRank，因为 Lenta.ru 新闻搜索不是网页链接图排名任务。  
但你的项目和 TF-IDF/BM25 很相关。

BM25 可以看作比 TF-IDF 更适合搜索排序的模型。它考虑：

```text
词频
词的稀有程度
文档长度归一化
```

所以如果老师问 TF-IDF，你可以顺便说：

```text
在我的项目中使用的是 BM25，它和 TF-IDF 思想相近，但更适合搜索排序。
```
