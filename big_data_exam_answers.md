## 1. Схожесть данных и уменьшение размерности данных

### 题目对应范围

Схожесть данных и уменьшение размерности данных. Понятие локально-чувствительного хэширования; подписи (хэши) множеств. Характеристическая матрица для множеств. Группы (bucket). Метрики схожести: сходство и расстояние по Жаккару, расстояние Левенштейна (редактирования), расстояние по Хэммингу, Евклидово расстояние, коэффициент Отиаи (косинусный). Вычисление оценок в лемме Джонсона-Линденштрауса.

### 中文解释分析

这一题考察的是大数据中如何判断对象之间是否相似，以及如何在数据维度很高时降低计算复杂度。

在大数据场景中，经常需要比较大量对象之间的相似性。例如：

- 判断两篇文章是否相似
- 判断两个用户行为是否相似
- 判断两张图片或两个信号是否相似
- 在大量文档中找出和查询最接近的文档

如果有 `n` 个对象，直接两两比较的复杂度通常接近：

```text
O(n^2)
```

当 `n` 很大时，这种方法不可行。因此需要：

- 相似性度量
- 降维
- 近似搜索
- 局部敏感哈希 LSH

局部敏感哈希 LSH 的核心思想是：

```text
相似对象更可能被映射到同一个 bucket；
不相似对象更不容易被映射到同一个 bucket。
```

普通哈希希望不同对象尽量均匀分散，而 LSH 希望相似对象保留在一起。

特征矩阵用于表示集合。矩阵中：

- 行表示所有可能的元素或特征
- 列表示不同的集合或文档
- `1` 表示该集合包含该元素
- `0` 表示不包含

例如：

```text
元素      S1   S2   S3
a         1    0    1
b         1    1    0
c         0    1    1
d         1    0    0
```

常见的相似性或距离度量包括：

1. Jaccard similarity：集合交集与并集的比例。
2. Jaccard distance：`1 - Jaccard similarity`。
3. Levenshtein distance：编辑距离，表示一个字符串变成另一个字符串需要的最少插入、删除、替换次数。
4. Hamming distance：两个等长字符串或向量中不同位置的数量。
5. Euclidean distance：欧几里得距离，表示数值向量之间的几何距离。
6. Otsiai coefficient / Cosine similarity：向量夹角余弦，常用于文本向量。

Johnson-Lindenstrauss lemma 说明，高维空间中的点可以随机投影到较低维空间，同时大致保持点之间的距离。这是随机投影和降维方法的重要理论基础。

### Русский официальный ответ

Схожесть данных и уменьшение размерности являются важными задачами анализа больших данных. При большом числе объектов прямое попарное сравнение становится слишком дорогим, поэтому используются метрики схожести, методы уменьшения размерности и приближенные методы поиска похожих объектов.

Локально-чувствительное хэширование, или LSH, — это семейство хэш-функций, при котором похожие объекты с высокой вероятностью попадают в одну и ту же группу, то есть bucket, а непохожие объекты попадают в одну группу с меньшей вероятностью. Это позволяет сначала находить кандидатов на похожие объекты, а затем сравнивать только их.

Для множеств используется характеристическая матрица. В ней строки соответствуют элементам или признакам, столбцы соответствуют множествам, а значения `1` и `0` показывают, входит ли элемент в данное множество.

Для оценки схожести используются разные метрики. Для множеств часто используется сходство по Жаккару:

```text
J(A, B) = |A ∩ B| / |A ∪ B|
```

Расстояние по Жаккару определяется так:

```text
D(A, B) = 1 - J(A, B)
```

Для строк используются расстояние Левенштейна и расстояние Хэмминга. Расстояние Левенштейна показывает минимальное число операций вставки, удаления и замены, необходимых для преобразования одной строки в другую. Расстояние Хэмминга показывает число различающихся позиций в двух строках одинаковой длины.

Для числовых векторов используется Евклидово расстояние. Для текстовых и векторных представлений часто используется коэффициент Отиаи, то есть косинусная мера:

```text
cos(a, b) = (a · b) / (|a| |b|)
```

Лемма Джонсона-Линденштрауса показывает, что множество точек в пространстве высокой размерности можно случайно спроецировать в пространство меньшей размерности так, чтобы расстояния между точками сохранялись приблизительно. Это используется для уменьшения размерности, ускорения поиска и снижения вычислительных затрат.

### 中文对照

数据相似性和降维是大数据分析中的重要问题。对象数量很大时，直接两两比较成本太高，所以需要使用相似性度量、降维方法和近似搜索。

LSH 是一种局部敏感哈希方法，它使相似对象有更高概率进入同一个 bucket，而不相似对象进入同一个 bucket 的概率较低。这样可以先找到候选相似对象，再做精确比较。

集合可以用特征矩阵表示。Jaccard 用于集合相似度，Levenshtein 和 Hamming 用于字符串，Euclidean 用于数值向量，cosine/Otsiai 用于向量方向相似度。

Johnson-Lindenstrauss 引理说明，高维数据可以随机投影到低维，同时近似保持距离。

## 2. Хэширование и сходство по Жаккару

### 题目对应范围

Хэширование и сходство по Жаккару. Сложность вычисления пересечения множеств. Алгоритм независимых по минимуму перестановок MinHash локально-чувствительного хэширования: вычисление хэша hmin(S) множества S, вычисление сигнатуры Hmin k независимых хэш-функций множества S, случайные перестановки, h(perm(x)) для x ∈ S, величина ошибки и вероятность близости.

### 中文解释分析

这一题的核心是 MinHash。Jaccard 相似度需要计算集合的交集和并集。如果集合很多、每个集合又很大，直接计算会非常慢。

MinHash 是一种用于估计 Jaccard 相似度的 LSH 方法。它不直接计算完整交集和并集，而是使用随机排列或哈希函数给集合生成短签名。

MinHash 的重要性质是：

```text
P(hmin(A) = hmin(B)) = J(A, B)
```

意思是：

```text
两个集合 MinHash 值相等的概率，等于它们的 Jaccard 相似度。
```

对于集合 `S`，`hmin(S)` 表示在某个随机排列下，集合 `S` 中最先出现的元素。

由于一个哈希值误差较大，实际会使用多个独立哈希函数，形成签名：

```text
Hmin(S) = [h1(S), h2(S), ..., hk(S)]
```

两个集合签名中相同位置的比例，可以作为 Jaccard 相似度的估计值。`k` 越大，估计越准确，但计算和存储成本也越高。

### Русский официальный ответ

Прямое вычисление сходства по Жаккару требует вычисления пересечения и объединения множеств. При большом числе множеств это становится вычислительно дорогой задачей. Поэтому для приближенной оценки сходства используется алгоритм MinHash.

MinHash — это метод локально-чувствительного хэширования для множеств. Для случайной перестановки строк характеристической матрицы значение `hmin(S)` множества `S` определяется как первая строка в этой перестановке, в которой множество `S` имеет значение `1`.

Главное свойство MinHash состоит в следующем:

```text
P(hmin(A) = hmin(B)) = J(A, B)
```

То есть вероятность того, что MinHash двух множеств совпадет, равна их сходству по Жаккару.

На практике используется несколько независимых хэш-функций. Для множества `S` строится сигнатура:

```text
Hmin(S) = [h1(S), h2(S), ..., hk(S)]
```

Оценка сходства двух множеств вычисляется как доля совпадающих элементов в их сигнатурах. Чем больше число хэш-функций `k`, тем меньше ошибка оценки, но тем выше вычислительная сложность и объем хранения.

Случайные перестановки в теории можно реализовывать с помощью хэш-функций, например через значения `h(perm(x))` для элементов `x ∈ S`. Это позволяет не хранить реальные перестановки больших матриц, а вычислять приближенные значения хэша.

### 中文对照

直接计算 Jaccard 相似度需要求集合交集和并集，在大数据中很慢。MinHash 用随机排列或哈希函数近似估计 Jaccard 相似度。

`hmin(S)` 是某个随机排列下集合中最先出现的元素。MinHash 的核心性质是两个集合哈希值相等的概率等于它们的 Jaccard 相似度。

实际中使用多个哈希函数组成签名，通过签名中相同位置的比例估计相似度。哈希函数越多，误差越小，但计算成本越高。

## 3. Инструменты вычисления локально-чувствительных хэшей

### 题目对应范围

Инструменты вычисления локально-чувствительных хэшей. Описание локально-чувствительного хэширования. Практические инструменты, метрики схожести, BucketedRandomProjectionLSH или MinHashLSH в PySpark.

### 中文解释分析

这一题要求你不仅知道 LSH 理论，还要知道 PySpark 中的实际工具。主要有两个：

- `MinHashLSH`
- `BucketedRandomProjectionLSH`

`MinHashLSH` 适合集合数据和 Jaccard 距离，例如文本 shingles、相似文档查找、去重。

`BucketedRandomProjectionLSH` 适合数值向量和欧几里得距离。它通过随机投影把向量映射到 bucket，距离近的向量更可能进入同一个 bucket。

选择哪个工具，取决于数据表示和距离度量。

### Русский официальный ответ

В PySpark существуют практические инструменты для локально-чувствительного хэширования. Основные из них — `MinHashLSH` и `BucketedRandomProjectionLSH`.

`MinHashLSH` используется для объектов, представленных как множества или разреженные векторы. Он основан на сходстве по Жаккару и подходит для поиска похожих документов, удаления дубликатов и сравнения множеств признаков.

`BucketedRandomProjectionLSH` используется для числовых векторов и Евклидова расстояния. Метод основан на случайной проекции: векторы проецируются на случайное направление, а затем попадают в buckets в зависимости от значения проекции. Близкие векторы с большей вероятностью попадают в один bucket.

Таким образом, выбор инструмента зависит от типа данных и метрики. Для множеств и расстояния Жаккара подходит `MinHashLSH`, а для числовых векторов и Евклидова расстояния подходит `BucketedRandomProjectionLSH`.

### 中文对照

PySpark 中有实际可用的 LSH 工具。`MinHashLSH` 用于集合和 Jaccard 距离，适合相似文本查找和去重。`BucketedRandomProjectionLSH` 用于数值向量和欧几里得距离，适合向量空间中的近似最近邻搜索。

## 4. Понятие больших данных

### 题目对应范围

Понятие больших данных. Признаки больших данных. Численные и маркетинговые характеристики больших данных, их промышленное применение. Большие данные в космических исследованиях. Понятие локально-чувствительного хэширования и подписей (хэшей) множеств. Концепция Map-Reduce и диспетчеризации запросов.

### 中文解释分析

这一题考察大数据的基本概念、应用场景以及 MapReduce 的基本思想。

大数据通常用 `5V` 描述：

- `Volume`：数据量大
- `Velocity`：数据产生和处理速度快
- `Variety`：数据类型多样
- `Veracity`：数据质量不稳定
- `Value`：可以从数据中提取价值

大数据可以用于搜索、推荐、金融风控、社交网络分析、工业监控、空间研究等。

空间研究中的大数据包括：

- 卫星图像
- 遥感数据
- 天文观测数据
- 宇宙射线探测数据
- 空间站传感器数据

MapReduce 是大数据处理的基本模型：

```text
input -> map -> shuffle -> reduce -> output
```

Map 阶段并行处理数据，Reduce 阶段合并结果。

### Русский официальный ответ

Большие данные — это данные, которые имеют большой объем, высокую скорость поступления, разнообразную структуру и требуют специальных распределенных методов хранения и обработки.

Обычно выделяют признаки больших данных, которые называют `5V`:

- `Volume` — большой объем данных
- `Velocity` — высокая скорость поступления и обработки
- `Variety` — разнообразие типов данных
- `Veracity` — неоднородность и возможная неточность данных
- `Value` — ценность, которую можно извлечь из данных

Большие данные применяются в поисковых системах, рекомендательных системах, финансовом анализе, социальных сетях, промышленности, медицине и космических исследованиях.

В космических исследованиях большие данные возникают при обработке спутниковых снимков, данных дистанционного зондирования, данных телескопов, сигналов радиотелескопов, данных космических аппаратов и измерений на МКС.

Для обработки больших данных используется концепция MapReduce. На этапе `Map` входные данные делятся на части и обрабатываются параллельно. На этапе `Reduce` промежуточные результаты группируются и объединяются в итоговый результат. Диспетчеризация запросов отвечает за распределение задач между узлами системы.

### 中文对照

大数据是数据量大、速度快、类型多、质量不稳定，并且需要从中提取价值的数据。它用于搜索、推荐、金融、工业、医学和空间研究等领域。

MapReduce 是分布式处理模型：Map 并行处理输入数据，Reduce 合并中间结果。调度器负责把任务分发到不同节点。

## 5. Хранение и анализ больших данных

### 题目对应范围

Хранение и анализ больших данных. Понятие распределенного хранилища. HDFS в Apache Hadoop. Сериализация данных. CSV, TSV, Apache ORC и Parquet. Typed bytes. Hadoop Streaming. Диспетчеризация и синхронизация запросов, консолидация результатов.

### 中文解释分析

这一题考察大数据如何存储、如何序列化，以及 Hadoop Streaming 的基本概念。

分布式存储是指数据不是放在一台机器上，而是分布在多台机器上。HDFS 是 Hadoop 的分布式文件系统。

HDFS 中：

- `NameNode` 保存元数据
- `DataNode` 保存真实数据块
- 大文件被切成多个 block
- block 可以有多个副本

常见数据格式：

- `CSV`：逗号分隔文本
- `TSV`：制表符分隔文本
- `Parquet`：列式存储，适合分析
- `ORC`：列式存储，常用于 Hive
- `Typed bytes`：Hadoop Streaming 中传递不同类型数据的二进制格式

Hadoop Streaming 允许用 Python 等语言编写 mapper 和 reducer。

### Русский официальный ответ

Для хранения больших данных используются распределенные хранилища. Одним из основных примеров является HDFS — Hadoop Distributed File System.

В HDFS большие файлы делятся на блоки, которые распределяются по разным узлам кластера. `NameNode` хранит метаданные о файлах и блоках, а `DataNode` хранит сами блоки данных. HDFS хорошо подходит для хранения больших файлов и последовательного чтения, но плохо подходит для большого количества маленьких файлов и частых изменений.

Для хранения и передачи данных используются разные форматы. `CSV` и `TSV` — простые текстовые форматы. `Parquet` и `ORC` — колонночные форматы, которые хорошо подходят для аналитических запросов, потому что позволяют читать только нужные столбцы и эффективно сжимать данные.

`Typed bytes` — это бинарный формат для Hadoop Streaming, который позволяет передавать значения разных типов. Hadoop Streaming позволяет писать mapper и reducer на языках вроде Python.

В распределенной обработке также важны диспетчеризация запросов, синхронизация задач и консолидация результатов. Диспетчеризация распределяет задачи между узлами, синхронизация обеспечивает правильный порядок этапов обработки, а консолидация объединяет результаты в итоговый ответ.

### 中文对照

HDFS 是 Hadoop 的分布式文件系统。它把大文件切成 block，分布在多个 DataNode 上，NameNode 管理元数据。HDFS 适合大文件顺序读写，不适合大量小文件和频繁修改。

CSV 和 TSV 是文本格式。Parquet 和 ORC 是列式格式，适合分析查询。Typed bytes 是 Hadoop Streaming 中传递不同数据类型的二进制格式。Hadoop Streaming 可以用 Python 写 mapper 和 reducer。

## 6. Концепции БД в анализе больших данных

### 题目对应范围

Концепции БД в анализе больших данных. Требования ACID к обычным БД. Понятия хранения строк (SQL) и хранения столбцов (Parquet/ORC) в БД. Примеры баз данных NOSQL и цель их использования. Распределенные БД и требования BASE к NOSQL (доступность, гибкость, согласованность в итоге). Анализ больших данных через SparkSQL. Эвристическая теорема Брюера CAP.

### 中文解释分析

这一题考察传统数据库和大数据/NoSQL 数据库的区别。

ACID 是传统数据库事务要求：

- `Atomicity`：原子性
- `Consistency`：一致性
- `Isolation`：隔离性
- `Durability`：持久性

BASE 是分布式 NoSQL 系统常见思想：

- `Basically Available`：基本可用
- `Soft state`：软状态
- `Eventually consistent`：最终一致

CAP 定理说明，分布式系统无法同时完全满足：

- `Consistency`：一致性
- `Availability`：可用性
- `Partition tolerance`：分区容错

行存储适合事务，列存储适合分析。NoSQL 用于高扩展性、高可用性和灵活数据结构。SparkSQL 用于大规模结构化数据分析。

### Русский официальный ответ

В классических реляционных базах данных важны требования ACID:

- `Atomicity` — атомарность
- `Consistency` — согласованность
- `Isolation` — изолированность
- `Durability` — долговечность

Эти свойства важны для транзакций, например в банковских системах.

В распределенных NoSQL-системах часто используется подход BASE:

- `Basically Available` — базовая доступность
- `Soft state` — мягкое состояние
- `Eventually consistent` — согласованность в итоге

Теорема CAP утверждает, что распределенная система не может одновременно полностью гарантировать согласованность, доступность и устойчивость к разделению сети:

- `Consistency`
- `Availability`
- `Partition tolerance`

На практике при наличии сетевых разделений система выбирает компромисс между согласованностью и доступностью.

Строковое хранение удобно для транзакционных систем, где часто читаются и обновляются целые строки. Колонночное хранение, например Parquet и ORC, удобно для аналитики, агрегаций и чтения отдельных столбцов.

NoSQL базы данных используются для масштабируемости, высокой доступности и гибкой схемы данных. Примеры NoSQL: MongoDB, Cassandra, HBase, Redis. SparkSQL позволяет анализировать большие структурированные данные с помощью SQL и DataFrame API.

### 中文对照

ACID 是传统数据库事务的四个要求：原子性、一致性、隔离性、持久性。BASE 是分布式 NoSQL 系统的思想：基本可用、软状态、最终一致。

CAP 定理说明分布式系统不能同时完全满足一致性、可用性和分区容错。现实系统通常需要在一致性和可用性之间权衡。

行存储适合事务，列存储适合分析。NoSQL 适合扩展性、高可用和灵活结构。SparkSQL 用于结构化大数据分析。

## 10. Локально-чувствительное хэширование - метод случайных проекций

### 题目对应范围

Локально-чувствительное хэширование - метод случайных проекций. Построение хэш-таблиц. Случайная проекция, Лемма Джонсона-Линденштрауса и расстояние Евклида. BucketedRandomProjectionLSH в PySpark.

### 中文解释分析

这一题是 LSH 的随机投影方法。

随机投影用于数值向量。核心思想是把高维向量投影到随机方向上：

```text
x -> x · r
```

其中 `r` 是随机向量。

如果两个点在原空间中距离近，那么投影后通常仍然比较接近。

BucketedRandomProjectionLSH 会把投影值按照 bucketLength 分桶：

```text
hash(x) = floor((x · r) / bucketLength)
```

它适用于欧几里得距离。

Johnson-Lindenstrauss lemma 为随机投影提供理论依据，说明高维点可以投影到低维，同时近似保持距离。

### Русский официальный ответ

Метод случайных проекций является одним из вариантов локально-чувствительного хэширования для числовых векторов и Евклидова расстояния.

Идея случайной проекции состоит в том, что вектор высокой размерности проецируется на случайное направление:

```text
x -> x · r
```

где `r` — случайный вектор. Если два объекта близки в исходном пространстве, то с высокой вероятностью их проекции также будут близки.

В `BucketedRandomProjectionLSH` значения проекций разбиваются на интервалы, то есть buckets. Объекты, попавшие в один bucket, считаются кандидатами на похожие объекты.

Одна из возможных форм хэша:

```text
hash(x) = floor((x · r) / bucketLength)
```

Лемма Джонсона-Линденштрауса дает теоретическое обоснование случайных проекций. Она утверждает, что множество точек в пространстве высокой размерности можно спроецировать в пространство меньшей размерности с небольшим искажением расстояний.

### 中文对照

随机投影是 LSH 的一种方法，适用于数值向量和欧几里得距离。它把高维向量投影到随机方向上，距离近的对象投影后仍然较可能接近。

BucketedRandomProjectionLSH 把投影值分桶，落入同一 bucket 的对象作为候选相似对象。Johnson-Lindenstrauss 引理说明这种随机投影可以近似保持距离。

## 11. PageRank и TF-IDF

### 题目对应范围

PageRank и TF-IDF. Вычисление PageRank, матричные преобразования, выявление ключевых слов.

### 中文解释分析

这一题包括 PageRank 和 TF-IDF。

PageRank 是网页重要性排序算法。核心思想是：

```text
一个网页重要，如果很多重要网页链接到它。
```

网页可以看成有向图：

- 节点是网页
- 边是链接

PageRank 可以通过迭代或矩阵乘法计算。

TF-IDF 用于关键词提取和文本检索。它衡量一个词对某篇文档的重要性。

TF 表示词在文档中出现频率。IDF 表示词在整个语料中的稀有程度。

一个词在当前文档中出现多，但在整个语料中不常见，那么它的 TF-IDF 分数高。

### Русский официальный ответ

PageRank — это алгоритм оценки важности страниц в графе ссылок. Основная идея состоит в том, что страница считается важной, если на нее ссылаются другие важные страницы.

Веб можно представить как ориентированный граф, где вершины — это страницы, а ребра — ссылки между страницами.

Упрощенная формула PageRank:

```text
PR(u) = sum(PR(v) / L(v))
```

где `v` — страницы, которые ссылаются на `u`, а `L(v)` — число исходящих ссылок со страницы `v`.

С учетом damping factor формула имеет вид:

```text
PR(u) = (1 - d) / N + d * sum(PR(v) / L(v))
```

Обычно `d = 0.85`.

PageRank можно вычислять итеративно как матричное преобразование:

```text
r_new = M * r_old
```

TF-IDF используется для выявления ключевых слов и оценки важности термина в документе.

TF означает `term frequency`, то есть частоту слова в документе.

IDF означает `inverse document frequency`:

```text
idf(t) = log(N / df(t))
```

где `N` — число документов, а `df(t)` — число документов, содержащих термин `t`.

Итоговая формула:

```text
tfidf(t, d) = tf(t, d) * idf(t)
```

Смысл TF-IDF состоит в том, что слово важно для документа, если оно часто встречается в этом документе, но редко встречается во всей коллекции.

### 中文对照

PageRank 是网页重要性排序算法。网页可以表示成有向图，节点是网页，边是链接。一个网页如果被很多重要网页链接，它也会变得重要。PageRank 可以用矩阵迭代计算。

TF-IDF 用于关键词提取和文本检索。TF 表示词在文档中的频率，IDF 表示词在整个语料中的稀有程度。一个词在当前文档中频繁出现、但在整个语料中不常见，则它对该文档更重要。
