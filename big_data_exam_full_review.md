## 2. 数据相似性与降维

### 题目原文

```text
Схожесть данных и уменьшение размерности данных.
Понятие локально-чувствительного хэширования; подписи множеств.
Характеристическая матрица для множеств. Bucket.
Метрики схожести: Жаккар, Левенштейн, Хэмминг, Евклидово расстояние, коэффициент Отиаи.
Лемма Джонсона-Линденштрауса.
```

### 导师视角解释

这道题问的是：大数据里对象很多，怎么判断两个对象是否相似？  
例如：

```text
两篇新闻是否相似
两个用户行为是否相似
一个 query 和一篇新闻是否相似
两张图片是否相似
```

如果有 `n` 个对象，两两比较需要接近：

```text
O(n^2)
```

当 `n` 很大时非常慢，所以需要：

1. 相似性度量。
2. 降维。
3. 近似搜索。
4. LSH。

### 俄语正式回答

```text
Схожесть данных и уменьшение размерности являются важными задачами анализа больших данных.
Когда объектов очень много, прямое попарное сравнение становится слишком дорогим по времени.
Поэтому используются метрики схожести, методы уменьшения размерности и приближенные методы поиска похожих объектов.

Локально-чувствительное хэширование, или LSH, — это семейство хэш-функций, при котором похожие объекты с высокой вероятностью попадают в один и тот же bucket, а непохожие объекты попадают в один bucket с меньшей вероятностью.
Это позволяет сначала находить небольшое число кандидатов, а затем сравнивать только их.

Для множеств можно использовать характеристическую матрицу. В такой матрице строки соответствуют элементам, столбцы соответствуют множествам, а значения 1 и 0 показывают, входит ли элемент в данное множество.

Для оценки схожести используются разные метрики.
Сходство Жаккара для множеств определяется как отношение размера пересечения к размеру объединения:
J(A, B) = |A ∩ B| / |A ∪ B|.
Расстояние Жаккара равно 1 - J(A, B).

Расстояние Левенштейна показывает минимальное число вставок, удалений и замен, необходимых для превращения одной строки в другую.
Расстояние Хэмминга показывает число различающихся позиций в двух строках одинаковой длины.
Евклидово расстояние используется для числовых векторов.
Коэффициент Отиаи, или косинусная мера, показывает близость направлений двух векторов:
cos(a, b) = (a · b) / (|a| |b|).

Лемма Джонсона-Линденштрауса утверждает, что множество точек в пространстве высокой размерности можно случайно спроецировать в пространство меньшей размерности так, чтобы расстояния между точками сохранялись приблизительно.
Это является теоретической основой случайных проекций и некоторых методов LSH.
```

### 中文对照

数据相似性和降维是大数据分析的重要问题。对象很多时，直接两两比较太慢，所以需要相似性度量、降维和近似搜索。  
LSH 是局部敏感哈希，它让相似对象更可能进入同一个 bucket，不相似对象不容易进入同一个 bucket。  
Jaccard 用于集合，Levenshtein 和 Hamming 用于字符串，Euclidean 用于数值向量，Cosine/Otsiai 用于向量方向相似性。  
Johnson-Lindenstrauss 引理说明，高维点可以随机投影到低维空间，同时近似保持距离。

## 3. MinHash 和 Jaccard 相似度

### 题目原文

```text
Хэширование и сходство по Жаккару.
Сложность вычисления пересечения множеств.
MinHash LSH: hmin(S), сигнатура Hmin, k независимых хэш-функций, случайные перестановки, величина ошибки и вероятность близости.
```

### 导师视角解释

这题问的是：如果对象可以表示成集合，怎么快速估计集合相似度？

例如两篇新闻可以表示成词集合：

```text
A = {футбол, чемпионат, россия}
B = {футбол, матч, россия}
```

它们有共同词，所以相似。

直接计算 Jaccard 要算交集和并集。如果集合很多，比较会很慢。MinHash 的作用就是：不用完整计算交集/并集，也能近似估计 Jaccard。

### 核心公式

```text
J(A, B) = |A ∩ B| / |A ∪ B|
D(A, B) = 1 - J(A, B)
```

MinHash 的关键性质：

```text
P(hmin(A) = hmin(B)) = J(A, B)
```

意思是：两个集合的 MinHash 值相等的概率，等于它们的 Jaccard 相似度。

### 俄语正式回答

```text
Сходство по Жаккару используется для сравнения множеств.
Оно определяется как отношение размера пересечения множеств к размеру их объединения:
J(A, B) = |A ∩ B| / |A ∪ B|.
Расстояние Жаккара равно:
D(A, B) = 1 - J(A, B).

Прямое вычисление сходства Жаккара требует вычисления пересечения и объединения множеств.
Если множеств много и они большие, такая операция становится дорогой.
Для приближенной оценки сходства используется MinHash.

MinHash — это метод локально-чувствительного хэширования для множеств.
Для случайной перестановки элементов значение hmin(S) множества S определяется как первый элемент множества S в этой перестановке.
Главное свойство MinHash:
P(hmin(A) = hmin(B)) = J(A, B).

На практике используется несколько независимых хэш-функций.
Для множества S строится сигнатура:
Hmin(S) = [h1(S), h2(S), ..., hk(S)].
Сходство двух множеств оценивается как доля совпадающих элементов их сигнатур.
Чем больше k, тем меньше ошибка оценки, но тем выше вычислительная стоимость и объем хранения.
```

### 中文对照

Jaccard 相似度用于比较集合，相似度等于交集大小除以并集大小。MinHash 是估计 Jaccard 的 LSH 方法。  
对于某个随机排列，`hmin(S)` 是集合 `S` 中最先出现的元素。MinHash 的核心性质是：两个集合 MinHash 相等的概率等于它们的 Jaccard 相似度。实际中使用多个哈希函数形成签名，通过签名相同位置的比例估计相似度。

### 和你的项目关系

你的主项目使用的是 Word2Vec + BM25，不是 MinHash。  
但是你可以这样说：

```text
Если представить новости как множества слов, можно было бы использовать MinHashLSH и сходство Жаккара.
В моем проекте документы представлены векторами Word2Vec, поэтому для LSH я использую BucketedRandomProjectionLSH, а не MinHashLSH.
```

中文：

```text

但我的项目把新闻表示成 Word2Vec 数值向量，所以更适合 BucketedRandomProjectionLSH。
```

## 4. PySpark 中的 LSH 工具

### 题目原文

```text
Инструменты вычисления локально-чувствительных хэшей.
BucketedRandomProjectionLSH или MinHashLSH в PySpark.
```

### 导师视角解释

老师不只问理论，还想知道你会不会用工具。重点是知道什么时候用哪个：

```text
MinHashLSH: 集合、Jaccard distance
BucketedRandomProjectionLSH: 数值向量、Euclidean distance
```

### 俄语正式回答

```text
В PySpark есть готовые инструменты для локально-чувствительного хэширования: MinHashLSH и BucketedRandomProjectionLSH.

MinHashLSH используется для объектов, представленных как множества или разреженные векторы.
Он основан на сходстве Жаккара и подходит для поиска похожих документов, удаления дубликатов и сравнения множеств признаков.

BucketedRandomProjectionLSH используется для числовых векторов и евклидова расстояния.
Метод основан на случайной проекции: векторы проецируются на случайное направление, затем значения проекции разбиваются на buckets.
Близкие векторы с большей вероятностью попадают в один bucket.

Выбор инструмента зависит от представления данных.
Если данные представлены как множества, удобно использовать MinHashLSH.
Если данные представлены как числовые векторы, лучше использовать BucketedRandomProjectionLSH.
```

### 中文对照

PySpark 提供两个常用 LSH 工具：`MinHashLSH` 和 `BucketedRandomProjectionLSH`。  
`MinHashLSH` 用于集合或稀疏向量，基于 Jaccard 相似度，适合文本去重、相似文档查找。  
`BucketedRandomProjectionLSH` 用于数值向量和欧氏距离，基于随机投影，把向量分配到 buckets。  
选择哪个工具取决于数据表示形式。

## 5. 什么是大数据

### 题目原文

```text
Понятие больших данных.

Концепция Map-Reduce и диспетчеризации запросов.
```

### 导师视角解释

这题问的是：什么样的数据叫大数据？为什么需要 Hadoop/Spark 这种系统？

常用解释是 Big Data 的几个 V：

```text
Volume: 数据量大
Velocity: 产生和处理速度快
Variety: 数据类型多
Veracity: 数据质量不稳定
Value: 要从数据中提取价值
```

### 俄语正式回答

```text
Большие данные — это данные такого объема, скорости поступления и разнообразия, что их трудно обрабатывать традиционными методами на одной машине.
Обычно признаки больших данных описываются через несколько V: volume, velocity, variety, veracity и value.

Volume означает большой объем данных.
Velocity означает высокую скорость появления и обработки данных.
Variety означает разнообразие форматов: текст, изображения, логи, таблицы, графы.
Veracity означает неоднородность и возможную неточность данных.
Value означает, что из данных нужно извлечь полезную информацию.

Для обработки больших данных используются распределенные системы, например Hadoop и Spark.
MapReduce позволяет разделить большую задачу на независимые части, выполнить их параллельно, а затем объединить результаты.
Диспетчеризация запросов означает распределение задач между узлами кластера и управление их выполнением.
```

### 中文对照

大数据是指数据量、速度和类型复杂到传统单机方法难以处理的数据。常用特征是 Volume、Velocity、Variety、Veracity、Value。  
为了处理大数据，需要 Hadoop、Spark 这样的分布式系统。MapReduce 把大任务拆成小任务并行处理，再合并结果。调度负责把任务分配给集群节点并管理执行。

## 6. HDFS、存储格式和 Hadoop Streaming

### 题目原文

```text
Хранение и анализ больших данных.
HDFS в Apache Hadoop. CSV, TSV, ORC, Parquet, typed bytes.
Hadoop Streaming. Консолидация результатов.
```

### 导师视角解释

这题问的是：大数据怎么存？为什么不是只放在普通文件夹里？各种格式有什么差别？

### HDFS 核心概念

```text
NameNode: 保存元数据
DataNode: 保存真实数据块
Block: 文件被切成的数据块
Replication: 副本机制
```

### 俄语正式回答

```text
HDFS, или Hadoop Distributed File System, — это распределенная файловая система Hadoop для хранения больших файлов на кластере машин.
Файл в HDFS делится на блоки, блоки распределяются по разным DataNode, а NameNode хранит метаданные: имена файлов, структуру каталогов и расположение блоков.
Для отказоустойчивости используется репликация блоков.

CSV и TSV — простые текстовые форматы, где значения разделены запятыми или табуляцией.
Они удобны для обмена данными, но не всегда эффективны для больших аналитических запросов.

Parquet и ORC — колоночные форматы хранения.
Они позволяют читать только нужные столбцы, хорошо сжимают данные и часто используются вместе со Spark и Hive.

Hadoop Streaming позволяет писать mapper и reducer на внешних языках, например Python или Bash.
Mapper читает данные из стандартного ввода и пишет промежуточные key-value пары в стандартный вывод.
Reducer получает сгруппированные данные и формирует итоговый результат.
Typed bytes — один из форматов передачи данных в Hadoop Streaming, который позволяет передавать не только строки, но и типизированные значения.
```

### 中文对照

HDFS 是 Hadoop 的分布式文件系统，文件被切成 block 存储在 DataNode 上，NameNode 保存元数据，并通过副本机制容错。  
CSV/TSV 简单但分析效率低。Parquet/ORC 是列式格式，适合大数据分析，因为可以只读需要的列并且压缩效果好。  
Hadoop Streaming 允许用 Python/Bash 写 mapper 和 reducer。typed bytes 是 Hadoop Streaming 中传递类型化数据的一种格式。

你的项目中：

## 7. ACID、BASE、CAP、NoSQL、SparkSQL

### 题目原文

```text
Концепции БД в анализе больших данных.
ACID, BASE, NoSQL, CAP, SparkSQL.
```

### 导师视角解释

这题问传统数据库和分布式数据库的区别。

ACID 关注强事务一致性。  
BASE 关注高可用和最终一致性。  
CAP 说明分布式系统不能同时完美满足一致性、可用性、分区容错。

### 俄语正式回答

```text
ACID — это набор требований к транзакциям в классических реляционных базах данных.
Он включает atomicity, consistency, isolation и durability.
Atomicity означает, что транзакция выполняется полностью или не выполняется вообще.
Consistency означает переход базы из одного корректного состояния в другое.
Isolation означает, что параллельные транзакции не мешают друг другу.
Durability означает, что после фиксации транзакции данные не теряются.

Он означает basically available, soft state и eventual consistency.
Такие системы могут не давать строгой согласованности в каждый момент, но обеспечивают доступность и приходят к согласованному состоянию со временем.

Теорема CAP утверждает, что распределенная система не может одновременно полностью гарантировать consistency, availability и partition tolerance.
При наличии сетевого разделения приходится выбирать компромисс между согласованностью и доступностью.

NoSQL базы данных используются для масштабируемого хранения больших и разнообразных данных.
SparkSQL позволяет выполнять SQL-подобные запросы поверх распределенных DataFrame в Spark.
```

CAP 定理说明分布式系统不能同时完全保证一致性、可用性、分区容错。发生网络分区时，通常要在一致性和可用性之间取舍。

## 10. 随机投影 LSH 和 Johnson-Lindenstrauss 引理

### 题目原文

```text
Локально-чувствительное хэширование - метод случайных проекций.
Построение хэш-таблиц. Случайная проекция.
Лемма Джонсона-Линденштрауса и расстояние Евклида.
BucketedRandomProjectionLSH в PySpark.
```

### 导师视角解释

这题和你的 LSH 扩展最相关。  
核心思想：对于数值向量，把高维向量投影到随机方向，再按区间分 bucket。

### 核心公式

随机投影：

```text
x -> x · r
```

分 bucket：

```text
hash(x) = floor((x · r) / bucketLength)
```

### 俄语正式回答

```text
Метод случайных проекций является одним из вариантов локально-чувствительного хэширования для числовых векторов и евклидова расстояния.

Идея случайной проекции состоит в том, что вектор высокой размерности проецируется на случайное направление:
x -> x · r,
где r — случайный вектор.
Если два объекта близки в исходном пространстве, то с высокой вероятностью их проекции также будут близки.

В BucketedRandomProjectionLSH значения проекций разбиваются на интервалы, то есть buckets.
Одна из возможных форм хэша:
hash(x) = floor((x · r) / bucketLength).
Объекты, попавшие в один или соседние buckets, рассматриваются как кандидаты на похожие объекты.

Лемма Джонсона-Линденштрауса дает теоретическое обоснование случайных проекций.
Она утверждает, что точки из пространства высокой размерности можно спроецировать в пространство меньшей размерности с небольшим искажением расстояний.

В PySpark этот метод реализован в BucketedRandomProjectionLSH.
Он подходит для данных, представленных числовыми векторами.
```

### 中文对照

随机投影 LSH 用于数值向量和欧氏距离。它把高维向量投影到随机方向，如果两个对象原本很近，它们投影后也大概率较近。  
BucketedRandomProjectionLSH 把投影值按 bucketLength 分桶。落入相同或接近 bucket 的对象作为候选相似对象。  
Johnson-Lindenstrauss 引理说明，高维点可以随机投影到低维空间，并近似保持距离。

你的 `find_lsh.py` 正好可以这样解释：

```text
Поэтому я использую BucketedRandomProjectionLSH, чтобы быстро найти кандидатов среди документов.

```

## 11. PageRank 和 TF-IDF

### 题目原文

```text
PageRank и TF-IDF.
Вычисление PageRank, матричные преобразования, выявление ключевых слов.
```

### 导师视角解释

这题有两个经典算法：

```text
PageRank: 图上的重要性排序
TF-IDF: 文本中的关键词权重
```

你的项目没有用 PageRank，但和 TF-IDF/BM25 很相关。

### PageRank 俄语回答

```text
PageRank — это алгоритм оценки важности страниц в графе ссылок.
Основная идея состоит в том, что страница считается важной, если на нее ссылаются другие важные страницы.

Веб можно представить как ориентированный граф, где вершины — это страницы, а ребра — ссылки.
Упрощенная формула PageRank:
PR(u) = sum(PR(v) / L(v)),
где v — страницы, которые ссылаются на u, а L(v) — число исходящих ссылок со страницы v.

С учетом damping factor формула имеет вид:
PR(u) = (1 - d) / N + d * sum(PR(v) / L(v)).
Обычно d = 0.85.
PageRank можно вычислять итеративно как матричное преобразование.
```

### TF-IDF 俄语回答

```text
TF-IDF используется для выявления ключевых слов и оценки важности термина в документе.
TF означает term frequency, то есть частоту слова в документе.
IDF означает inverse document frequency и показывает, насколько редко термин встречается во всей коллекции.

Формула IDF:
idf(t) = log(N / df(t)),
где N — число документов, а df(t) — число документов, содержащих термин t.

Итоговая формула:
tfidf(t, d) = tf(t, d) * idf(t).

Смысл TF-IDF состоит в том, что слово важно для документа, если оно часто встречается в этом документе, но редко встречается во всей коллекции.
```

### 中文对照

PageRank 用于网页链接图的重要性排序，一个网页如果被很多重要网页链接，它也会更重要。  
TF-IDF 用于关键词提取和文本检索。TF 表示词在当前文档中的频率，IDF 表示词在整个语料中的稀有程度。一个词在当前文档中频繁出现、但在整个语料中少见，则它更重要。

### 和你的项目关系

你的项目使用 BM25，不是直接 TF-IDF，但 BM25 与 TF-IDF 思想相关。

答辩句子：

```text
В моем проекте используется BM25.
Он близок по идее к TF-IDF, потому что учитывает частоту термина и редкость термина в коллекции.
Но BM25 дополнительно учитывает длину документа и лучше подходит для ранжирования в поисковых системах.
```

## 15. 项目完整流程

### 15.1 训练阶段

```text
HDFS news_lenta.csv
-> Spark 读取数据
-> 文本清洗
-> 分词
-> 去停用词
-> 训练 Word2Vec
-> 生成每篇新闻的 document_vector
-> 统计 BM25 需要的数据
-> 保存模型、索引、统计到 HDFS
```

俄语：

```text
На этапе обучения Spark читает новости из HDFS, очищает текст, выполняет токенизацию, удаляет стоп-слова, обучает Word2Vec, строит векторы документов, считает статистику BM25 и сохраняет модель, индекс и статистику в HDFS.
```

### 15.2 查询阶段

```text
用户输入 query
-> query 清洗和分词
-> 加载 Word2Vec 模型
-> query 转成向量
-> 加载文档索引
-> 计算 Word2Vec cosine similarity
-> 计算 BM25
-> final score
-> 输出 Top-10 新闻
```

俄语：

```text
На этапе поиска пользовательский запрос проходит такую же предобработку, как документы при обучении.
Затем запрос преобразуется в вектор Word2Vec.
Система сравнивает вектор запроса с векторами документов, считает BM25 и итоговый score, после чего выводит Top-10 наиболее релевантных новостей.
```

### 15.3 LSH 查询阶段

```text
用户输入 query
-> query 转成 Word2Vec 向量
-> BucketedRandomProjectionLSH 找候选新闻
-> 候选新闻上计算 Word2Vec + BM25
-> 输出 Top-10
```

俄语：

```text
В LSH-версии запрос сначала преобразуется в вектор Word2Vec.
BucketedRandomProjectionLSH находит ограниченное число кандидатов среди документов.
Затем только эти кандидаты ранжируются более точно с помощью Word2Vec cosine similarity и BM25.
```

## 16. 模型设计解释

### 16.1 Word2Vec 是什么

俄语：

```text
Word2Vec — это нейронная модель, которая обучает векторные представления слов.
Слова, которые встречаются в похожих контекстах, получают близкие векторы.
В проекте Word2Vec используется для получения векторов документов и запросов.
```

中文：

Word2Vec 是学习词向量的神经网络模型。上下文相近的词会得到相近向量。项目中用它生成文档向量和查询向量。

### 16.2 BM25 是什么

俄语：

```text
BM25 — это функция ранжирования в информационном поиске.
Она учитывает частоту термина в документе, редкость термина в коллекции и длину документа.
BM25 усиливает документы, где реально встречаются ключевые слова запроса.
```

中文：

BM25 是搜索排序函数，考虑词频、词的稀有程度和文档长度。它可以强化真正包含查询关键词的新闻。

### 16.3 为什么 Word2Vec + BM25

俄语：

```text
Word2Vec хорошо показывает семантическую близость, но может быть недостаточно чувствителен к точному совпадению ключевых слов.
BM25 хорошо учитывает точное наличие слов запроса, но не понимает семантическую близость.
Поэтому гибрид Word2Vec + BM25 дает более устойчивое ранжирование.
```

中文：

Word2Vec 擅长语义相似，但对关键词精确匹配不一定敏感。BM25 擅长关键词匹配，但不理解语义。所以组合后排序更稳。

### 16.4 LSH 是不是神经网络

俄语：

```text
LSH не является нейронной моделью.
Это метод приближенного поиска похожих объектов.
Нейронная модель в моем проекте — Word2Vec.
LSH используется отдельно как быстрый способ найти кандидатов.
```

中文：

LSH 不是神经网络。它是近似相似搜索方法。你的神经网络模型是 Word2Vec。LSH 只是用来快速找候选新闻。

## 36. 4 分必须会说的关键词

```text
LSH: похожие объекты попадают в один bucket с высокой вероятностью.
MinHash: P(hmin(A)=hmin(B)) = J(A,B).
Jaccard: |A ∩ B| / |A ∪ B|.
BucketedRandomProjectionLSH: для числовых векторов и евклидова расстояния.
HDFS: NameNode + DataNode + blocks + replication.
MapReduce: Map -> Shuffle -> Reduce.
Spark: распределенная обработка, DataFrame, MLlib, вычисления в памяти.
ACID: atomicity, consistency, isolation, durability.
BASE: basically available, soft state, eventual consistency.
CAP: нельзя одновременно полностью гарантировать C, A и P.
Parquet/ORC: колоночные форматы для аналитики.
TF-IDF: tf * idf.
BM25: похож на TF-IDF, но лучше для поискового ранжирования.
Word2Vec: нейронная модель для векторных представлений слов.
```
